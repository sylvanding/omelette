## Codebase Patterns
- Reference export endpoints use `/projects/{id}/export/{format}` pattern (bibtex, ris, zotero)
- Zotero integration requires `ZOTERO_API_KEY` and `ZOTERO_USER_ID` env vars; falls back to BibTeX preview when not configured
- `downloadExport()` in `bibliography-export.ts` handles client-side file downloads for BibTeX/RIS/EndNote
- Export dialog uses two-tab pattern (Download + Zotero) with paper preview

## Codebase Patterns
- Team member model uses `email` (not `user_id`) since the app is currently single-user; invites are tracked by email
- RBAC middleware uses `X-User-Email` header for identity; without it, single-user mode is assumed and access is granted
- API error responses use `{ code, message, data }` format (not FastAPI's default `{ detail }`)
- Error responses from HTTPException are wrapped by a custom exception handler in the main app
- Team members cascade-delete when their project is deleted (`ondelete="CASCADE"`)
