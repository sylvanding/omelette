## Codebase Patterns
- Backend uses `ApiResponse` envelope for all endpoints with `{code, message, data}` structure
- LLM calls use `llm.chat_json()` for JSON responses and `task_type` for mock routing
- New services go in `app/services/` with constructor taking `LLMClient`
- Each route module exports `router = APIRouter(tags=["..."])` — mounted via `__init__.py`
- Frontend API calls use axios instance from `@/lib/api` that unwraps `response.data`
- Mock responses live in `app/services/llm/adapters/mock_adapter.py` — add task_type entries
- Test fixtures use `client` (AsyncClient), `project_id`, and `setup_db` from conftest.py
- Frontend uses `@tanstack/react-query` for data fetching with `useQuery`
- shadcn/ui components may not exist for everything — use native HTML elements when missing (e.g., `<input type="range">` instead of missing Slider)
