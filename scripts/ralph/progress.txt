## Codebase Patterns
- UI components in `src/components/ui/` use radix-ui primitives via `radix-ui` package (follow `select.tsx`, `popover.tsx` patterns)
- API service layer in `src/services/api.ts` — axios interceptor unwraps `response.data`, use raw `fetch` for blob downloads
- Backend endpoints in `backend/app/api/v1/` — use `ApiResponse<T>` wrapper, `FileResponse`/`Response` for file downloads
- i18n keys nested under feature name in both `en.json` and `zh.json` (e.g., `papers.export`)
- Tests use vitest with jsdom, fixtures in `src/test/fixtures/`
- Toolbar components receive props from parent page (PapersPage → PapersToolbar → PapersExportDropdown)
- `usePapersColumns` hook in `papers-columns.tsx` accepts callbacks for inline actions (rating change, delete, etc.)
- Backend quality_tags filter uses PostgreSQL `overlap()` for array matching — pass comma-separated tags in query param
- Alembic migration chain may be broken — set `down_revision = None` if prior migration files are missing
- New model files go in `backend/app/models/` with `Mapped[T]` + `mapped_column` pattern, register in `models/__init__.py` and add relationship on `Project`
- New API routers in `backend/app/api/v1/` — register in `api/v1/__init__.py` with `include_router`
- radix-ui SelectItem cannot have empty string value — use `"all"` or similar placeholder instead of `""`
