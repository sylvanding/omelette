## Codebase Patterns
- Browser extension goes in `browser-extension/` at project root, separate from backend/frontend
- Use Query params for browser upload endpoint (not JSON body) — simpler for extension GET/POST from popup
- `async_session_factory` (not `async_session`) is the correct import from `app.database`
- Browser upload endpoint path: `/projects/{id}/upload/browser` (separate from `/papers/upload`)
- Backend endpoints use FastAPI `APIRouter` with `Depends(get_db)` and `Depends(get_project)` for DI
- Services import `LLMClient` lazily inside endpoints via `get_llm()` from `app.api.deps`
- Request/response schemas are Pydantic models defined inline in API files
- Frontend API calls are organized in `src/services/api.ts` as namespace objects with `.then(r => r.data)` unwrapping
- Dialogs use fixed-position overlays with `fixed inset-0 z-50` pattern
- MSW mock handlers in `src/test/mocks/handlers.ts` need new endpoints added for tests
- `useToastMutation` is the standard mutation hook with toast + query invalidation
